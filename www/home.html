Webserv Project

HTTP Server Implementation

Project Overview

This HTTP server implementation is part of the 1337 cursus, designed to provide a deep understanding of web server architecture and HTTP protocol handling.

Technical Stack

  • C/C++ Implementation
  • Socket Programming
  • HTTP/1.1 Protocol
  • Configuration File Support

Supported Methods

  • GET - Retrieve resources
  • POST - Submit data
  • DELETE - Remove resources

Features

  • Multiple port listening
  • CGI script execution
  • Session & Cookies management
  • Custom error pages

Documentation

Configuration

The server uses a configuration file to define server blocks, locations, and various settings. The server block can handle multiple domains and ports.

Error Handling

Custom error pages are implemented for various HTTP status codes including 404 (Not Found), 500 (Internal Server Error), 403 (Forbidden), and others.

File Upload

The server supports file uploads through POST requests with multipart/form-data encoding. Uploaded files are stored in designated directories.
